SpecBooks Integration Guide

Queue Gateway contract for partner-facing requests

Authentication

All SpecBooks requests must include:

X-SPECBOOKS-API-KEY: {{specbook-api-key}}

Available Endpoints (10)

Endpoint Method + Path Description
Get Customer GET /api/specbooks/customers/{customerId} Fetch customer account details, including address/contact fields.
Get Opportunity GET /api/specbooks/opportunities/{opportunityId} Fetch an opportunity and its line items.
Create Opportunity POST /api/specbooks/opportunities Create a new opportunity with provided payload.
Create Stock Item POST /api/specbooks/stock-items Create a new stock item with required fields and optional additional stock item fields.
Update Opportunity PATCH /api/specbooks/opportunities/{opportunityId} Apply partial updates to an existing opportunity, including product line changes.
Poll Job GET /api/specbooks/jobs/{jobId} Check async job status/result (queued, processing, succeeded, failed).
Get Stock Item (Single) GET /api/specbooks/stock-items/{inventoryId} Fetch a single stock item by Inventory ID (case-insensitive).
Get Stock Items (Multiple) GET /api/specbooks/stock-items/{inventoryId}?inventoryIds={id1},{id2} Fetch multiple stock items by passing comma-separated Inventory IDs in inventoryIds.
Get Item Class GET /api/specbooks/item-classes/{itemClassId} Fetch item class metadata by class identifier.
Get Contact GET /api/specbooks/contacts/{contactId} Fetch a contact by contact identifier.

Endpoint 5: Poll Job Implemented

Purpose

Poll a queued job by jobId to check whether it is still processing or completed.

Request

  • Method: GET
  • URL: {{baseUrl}}/api/specbooks/jobs/{{jobId}}
  • Required Header: X-SPECBOOKS-API-KEY: {{specbook-api-key}}

Job Status Values

  • queued
  • processing
  • succeeded
  • failed

Response Note

The response payload is based on which endpoint the jobId came from (for example, Get Customer vs Get Opportunity vs Create/Update Opportunity).
